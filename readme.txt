1. Perbedaan tanggung jawab Front-End dan Back-End

Front-End bertugas mengatur tampilan dan interaksi yang
langsung dilihat serta digunakan oleh pengguna, seperti
form, tombol, teks, dan tata letak halaman.

Back-End bertugas mengolah data yang dikirim dari pengguna,
melakukan validasi, menjalankan logika aplikasi, dan
memberikan respon dari server.

Analogi toko:
Front-End diibaratkan sebagai kasir dan etalase toko yang
melayani pelanggan, sedangkan Back-End adalah gudang dan
sistem pencatatan yang mengatur stok dan transaksi.


2. Perbedaan method GET dan POST dalam pengiriman data form

Method GET mengirimkan data melalui URL sehingga data dapat
terlihat dan mudah diakses. Method ini cocok untuk data
yang tidak sensitif dan hanya untuk pengambilan informasi.

Method POST mengirimkan data melalui body request sehingga
data tidak terlihat di URL dan lebih aman. Method POST
sebaiknya digunakan saat mengirim data penting seperti
form kontak, login, atau data dengan jumlah besar.


3. Alasan validasi input dilakukan di server (Back-End)

Validasi input harus dilakukan di server karena validasi
di browser (Front-End) dapat dimatikan atau dilewati oleh
pengguna. Dengan validasi di server, data yang diterima
dipastikan sesuai aturan, aman, dan tidak merusak sistem
sebelum diproses lebih lanjut.
